# Установка зависимостей

```
pip install -r requirements.txt
```

# Настройка доступа к S3 для DVC

```
dvc remote modify storage access_key_id <your_access_key>
dvc remote modify storage secret_access_key <your_secret_key>
```

# Загрузка финального и промежуточного датасетов

```
dvc pull
```

Эта команда явялется аналогом `git pull` для датасетов. Поэтому выполняйте ее перед написанием нового кода. Так как что-то могло измениться

Теперь в папке `data/interim` появятся промежуточные датасеты, разделенные на этапы

**Важно!** нельзя использоваться  `data/processed/final.{csv, parquet}`. Используйте файлы из `data/interim` 

# Запуск пайплайна

Перед запуском нужно положить следующие файлы в директорию `secrets`, если планируется парсинг геоточек. *Скорее всего, в dvc уже лежат спаршенные геоточки, поэтому вам ничего не нужно добавлять в папку `secrets`. Просто выполните `dvc pull`.*

```bash
├── secrets
│   └── geocoding
│       └── ya_api_keys.csv
```

Запуск пайплайны (этапы, которые уже лежат на dvc пропустятся автоматически).

```
dvc repro
```

# Сохранение и отправка результатов работы пайплайна

```
dvc commit

dvc push
```

# Подробнее про dvc
Репозиторий используйет DVC (Data Version Control) для хранения и кэширования этапов пайплайна. Он нужен, чтобы 1) где-то хранить тяжелые файлы; 2) не перезапускать пайплайн с нуля (важно для дорогих по вычислениям этапов). DVC хранит данные на S3 bucket (который оплатил Глеб).
